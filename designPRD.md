2. Design PRD
Visual Identity: The app should look like a "Dev Tool"—minimalist, utilizing a monospace font for code blocks and a sleek dark mode by default.

Key Screens
Project Hub (Home): A list of previous projects the user has generated posts for.

The "Composer" (Main Feature):

Top section for text input/description.

Horizontal scroll for image uploads (Image Pickers).

Chip selectors for "Target Platform."

The "Preview" Screen: * A split-view showing the generated text alongside the uploaded images in a "post-like" mockup.

"Regenerate" button with a "Refine" text field.

User Experience (UX) Principles
Zero Friction: No login required for the first generation.

Visual Validation: Use a Markdown renderer so the user sees exactly how the Peerlist post will look.

Speed: Provide a "Loading Shimmer" that mimics "AI Thinking" to manage user expectations during API calls.
